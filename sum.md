# stemplay (band-mixer) 프로젝트 대화 요약

## 1. 최초 구축 (v1)

### 1-1. 프로젝트 최초 생성
**프롬프트:** 밴드 음악을 악기별로 볼륨 조절하며 감상할 수 있는 개인용 웹페이지를 Demucs로 만들어달라.

**수정 사항:**
- `server.py`, `index.html`, `requirements.txt`, `README.md` 신규 생성.
- 보안: 셸 명령 주입 위험 제거(파일명 직접 삽입 대신 리스트 인자 방식), 파일명 정제.
- same-origin 서빙으로 CORS 불필요하게 구조화, 프론트는 상대 경로 요청 사용.
- 일시정지 시 재생 위치 유지, 탐색(seek) 슬라이더·시간 표시 추가.
- 전송량 절감을 위해 WAV 대신 MP3 출력, 동일 곡 재업로드 시 캐싱으로 재분리 생략.

### 1-2. 설치/실행 스크립트 (setup.sh)
**프롬프트:** 서버 실행에 필요한 설정을 한 번에 처리하는 쉘 스크립트, 이후 서버 실행까지 포함해달라.

**수정 사항:**
- `setup.sh` 신규: 패키지 조건부 설치, 가상환경 생성, GPU 감지로 없으면 CPU 전용 torch 설치, 설치 후 서버까지 실행. `--setup-only` 옵션 추가.

### 1-3. 유튜브 링크 입력 방식 전환
**프롬프트:** 파일 업로드 대신 유튜브 링크로 음원을 받고 싶다. 서버는 백그라운드로 실행해달라.

**수정 사항:**
- `/upload` → `/process`(JSON `{url}`)로 교체, 영상 ID만 추출해 주소 재조립(임의 사이트 다운로드 차단).
- `yt-dlp` 의존성 추가, 다운로드 후 Demucs 분리.
- 프론트를 링크 입력창으로 변경, 곡 제목 표시.
- `setup.sh`를 포그라운드 실행 → 백그라운드(`nohup &`) 실행으로 변경, 헬스체크·로그 안내 추가.

### 1-4. 악기 자동 감지
**프롬프트:** 분리할 악기를 자동 감지해서 분리하고 싶다(어려우면 5트랙 고정으로).

**수정 사항:**
- 모델을 6-stem(`htdemucs_6s`)으로 교체, 각 트랙 음량을 측정해 무음 트랙은 감지 결과에서 제외(`meta.json` 캐싱).
- 프론트는 감지된 트랙만큼 슬라이더 동적 생성.
- `setup.sh`가 재실행 시 기존 서버 프로세스·포트 점유를 정리하도록 수정.

### 1-5. 서비스명 결정
**프롬프트:** 프로젝트/레포 이름을 추천해달라 → `stemplay`로 확정.

**수정 사항:** `index.html` 타이틀·헤더, `README.md` 제목에 "stemplay" 반영.

### 1-6. `.gitignore` 작성
**프롬프트:** 가상환경, 로그 등 깃허브에 올리면 안 되는 파일을 `.gitignore`에 정리해달라.

**수정 사항:** `.venv/`, `__pycache__/`, `*.log`, `uploads/`, `separated/`, 오디오 확장자, `.env`, 에디터 설정 등 제외 처리.

### 1-7. 음원 삭제 버튼
**프롬프트:** 클릭하면 서버의 업로드/분리 결과를 모두 삭제하는 버튼을 추가해달라.

**수정 사항:**
- `server.py`에 `POST /clear` 추가(디렉토리는 남기고 내용만 삭제).
- `index.html`에 확인창(confirm) 포함한 "음원 삭제" 버튼 추가.

## 2. EC2 배포 및 nginx/SSL 구성

### 2-1. 배포 스크립트 작성
**프롬프트:** README 기준 EC2 배포용 쉘 스크립트 작성 → 이후 도메인·SSL·nginx 유무에 따라 반복 수정.

**수정 사항:**
- `deploy.sh` 신규: 스왑 자동 생성(t2.micro OOM 방지), gunicorn을 systemd 서비스로 구동.
- 기존에 nginx·도메인·SSL을 보유한 것이 확인되면서 gunicorn 바인딩을 `0.0.0.0`→`127.0.0.1:5000`으로 변경(외부 접근은 nginx 경유).
- 스크립트를 `deploy-backend.sh` / `deploy-frontend.sh` / 통합 `deploy.sh` 3개로 분리. 프론트엔드 스크립트는 nginx 설정 파일 작성(정적 서빙 + API 프록시), 인증서 유무에 따라 HTTPS/HTTP 분기 생성.
- DNS/certbot 관련 문의는 코드 변경 없이 안내(레코드 확인, 발급 한도 등).

### 2-2. nginx 설정 보존 + SSL 안내 강화
**프롬프트:** 배포 시 이미 존재하는 nginx 설정 파일을 건드리지 않게, 그리고 SSL 설정 커맨드를 출력에 포함해달라.

**수정 사항:**
- `deploy-frontend.sh`: 설정 파일이 이미 있으면 건드리지 않고 보존(`--force`로 강제 재생성 가능), 없을 때만 신규 생성.
- 인증서 미적용 상태면 배포 출력에 certbot 설치·발급 커맨드를 표시하도록 변경.

## 3. v2 아키텍처 전환 — 서버/워커 분리 구조

### 3-1. 방향 결정
**프롬프트:** EC2에서 유튜브 다운로드가 계속 막힌다(IP 차단). EC2는 저장/서빙만, 실제 분리는 개인 PC에서 처리하도록 구조를 바꾸고 싶다.

**결정:** EC2에 작업 큐를 두고 집 PC 워커가 폴링(pull)하는 구조로 전환. 포트포워딩 불필요, 큐는 메모리로 처리, EC2에서 yt-dlp/Demucs/torch 제거.

### 3-2. README 선(先) 작성
**수정 사항:** 기존 README를 `README_v1.md`로 백업, 새 구조 기준 README 작성(흐름도, API 표, 인증 방식, 설치·운영 가이드).

### 3-3. 곡 리스트 화면 기획
**프롬프트:** 미리 분리된 곡 리스트를 보여주고, 곡 선택 시 믹서 화면으로 전환되게 하고 싶다.

**수정 사항:** README 스펙에 "링크 입력 + 곡 리스트" 구조, `GET /songs` API 명시.

### 3-4. 실제 구현 (server.py / worker.py / index.html / run.sh)
**프롬프트:** 확정된 순서대로 각 파일을 구현해달라(순차 요청).

**수정 사항:**
- **server.py** 전면 재작성: yt-dlp/Demucs 로직 제거, 메모리 기반 작업 큐(`queued`→`processing`→완료/`failed`, 무응답 워커 30분 후 재배정) 도입. 브라우저용 API(`/songs`, `/process`, `/status/<id>`)와 워커용 API(`/worker/job`, `/worker/result`, `/worker/fail`)를 토큰 인증(`X-Worker-Token`, 상수시간 비교)으로 분리. 업로드는 임시 폴더에 받아 완성 후 교체.
- **worker.py** 신규: 무한 루프 폴링 → 다운로드/분리/감지 → 업로드, 개별 작업 실패해도 워커 자체는 죽지 않고 다음 작업 진행. 로컬 캐시 재사용 로직 포함.
- **index.html** 재작성: 곡 리스트 화면(5초 폴링, 진행상태 아이콘, 워커 꺼짐 배너)과 믹서 화면(기존 멀티트랙 엔진 유지 + 곡 리스트로 복귀) 두 화면 전환 구조.
- **run.sh** 신규: `server`/`worker`/전체 역할 인자, 실행 시 기존 프로세스 정리 후 재시작, 로그 파일 분리, ffmpeg/node 자동 설치.

### 3-5. 실행법 문서화
**프롬프트:** 케이스별(EC2만/로컬만/EC2+로컬 혼합) 실행법을 README에 정리해달라.

**수정 사항:** README에 명령·환경변수 표와 케이스 A/B/C 구분, 종료 명령 별도 정리.

## 5. 보안 강화 — .env 분리, 접속 제어, 웹 인증 + nginx 분리

### 5-1. 토큰 .env 분리
**프롬프트:** 토큰 같은 값을 `.env`로 빼서 관리하고 `.gitignore`에 추가해달라.

**수정 사항:** `.env`/`.env.example` 신규 생성, `server.py`·`worker.py`·`run.sh`에 자체 `.env` 로더 추가(외부 패키지 미사용).

### 5-2. 접속 구조 문답 (코드 변경 없음)
집/회사 등 여러 IP에서 접속, 핸드폰 셀룰러 접속 가능 여부 등을 상담 → 보안그룹 IP 제한의 한계를 설명하고 이후 "전체 공개 + 웹 인증" 방향으로 결정.

### 5-3. 웹 인증 + nginx 서버 분리 구현
**프롬프트:** 웹 UI에 비밀번호 인증을 추가하고, 기존에 설치된 nginx와 기능 서버를 분리하는 스크립트를 작성해달라.

**수정 사항:**
- `server.py`: 비밀번호 기반 로그인(`/login`), `before_request` 가드, 세션 쿠키 인증 추가.
- `setup-nginx.sh` 신규: 정적 서빙/오디오 서빙은 nginx, API는 gunicorn 프록시로 분리하는 구성.
- `index.html`, `run.sh`, `.env`(`WEB_PASSWORD` 등) 관련 후속 수정.
- (이 세션은 사용량 한도로 중간에 종료되어 최종 검증 결과는 기록되지 않음.)

## 6. 운영 안정화 — 워커가 자꾸 꺼지는 문제 수정 (2026-07-21, 가장 최근)

**프롬프트:** EC2 서버는 정상인데 로컬 WSL 워커가 자꾸 꺼진다, 점검해달라 → "1, 2번만" 적용 요청.

**진단:** WSL2 VM 자체가 내려갔다 재기동된 정황(uptime 9분) + 자동 재시작 장치 부재 + JSON 파싱 코드가 `try` 밖에 있어 비정상 응답 시 워커가 죽는 잠재 버그.

**수정 사항:**
- `worker.py`: 작업 응답 JSON 파싱을 `try` 블록 안으로 이동(비정상 200 응답에도 죽지 않고 재시도).
- `~/.config/systemd/user/stemplay-worker.service` 신규 작성: `Restart=always`로 크래시/재부팅 시 자동 복구, 재시작 테스트 통과. `./run.sh worker`는 이후 사용 중지 안내.
- (미완: `sudo loginctl enable-linger` 필요, WSL VM 자체 종료 방지는 Windows `.wslconfig` 영역이라 별도 조치 필요.)
